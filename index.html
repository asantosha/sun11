    HTML (HyperText Markup Language)
  1. Element: The building blocks of an HTML document, defined by tags (e.g., < div >, < p >, < a >).
  2. Tag: The markup that defines an HTML element (e.g., < h1 >, < img >).
  3. Attribute: Provides additional information about an element (e.g., src, alt, href). DOM (Document Object Model): A programming interface for HTML documents.
  4. DOCTYPE: Declares the document type and version of HTML.
  5. Form: Used to collect user input (e.g., < form >, < input >).
  6. Table: Organizes data into rows and columns (< table >, < tr >, < td >).
  7. Semantic HTML: Tags that provide meaning to the content (e.g., < article >, < section >, < header >).
    8.

    CSS (Cascading Style Sheets)

  1. Selector: Specifies which HTML elements to style (e.g., .class, #id, element).
  2. Property: The aspect of the element you want to change (e.g., color, font-size, margin).
  3. The setting for a property (e.g., red, 16px, 10px).
  4. Box Model: Describes the rectangular boxes generated for elements (e.g., content, padding, border, margin).
  5. Flexbox: A layout model for distributing space along a single axis.
  6. Grid: A layout system for creating complex, responsive designs.
  7. Media Queries: Allows the application of styles based on device characteristics.
  8. Pseudo-classes: Styles applied under certain conditions (e.g., :hover, :first-child).
    9.

JavaScript
  • Variable: Stores data that can be referenced and manipulated (e.g., var, let, const).
  • Function: A block of code designed to perform a particular task (e.g., function myFunction() {}).
  • Event: Actions that occur in the browser (e.g., click, mouseover, load).
  • Object: A collection of properties and methods (e.g., { name: "John", age: 30 }).
  • Array: A list of values (e.g., [1, 2, 3]).
  • DOM Manipulation: Methods to dynamically change the content and structure of a web page (e.g., document.getElementById(), document.querySelector()).
  • Asynchronous Programming: Managing code that runs independently of the main program flow (e.g., setTimeout, fetch, async/await).
  • Closures: Functions that reference variables from their containing scope.

    Animation
  • Keyframes: Define the styles at various points in an animation sequence (@keyframes in CSS).
  • Transition: Smoothly changes property values over a specified duration (e.g., transition: all 0.5s).
  • Transform: Applies 2D or 3D transformations to an element (e.g., rotate, scale, translate).
  • Easing: Controls the acceleration and deceleration of the transition effect (e.g., ease, linear, ease-in).
  • FPS (Frames Per Second): Measures the frame rate of an animation.
  • SVG (Scalable Vector Graphics): XML-based vector image format for defining graphics and animations.
  • Canvas: An HTML element used to draw graphics via scripting (usually JavaScript).
  • Library: Pre-written code that facilitates animations (e.g., GreenSock, Anime.js).
  • These terms cover fundamental concepts and tools you'll encounter when working with HTML, CSS, JavaScript, and web animations
